# Codeg Mobile Android 安装与配对

## 使用条件

- Android 7.0（API 24）或更高版本。
- 电脑上的 Codeg 必须保持运行并可访问互联网。
- Relay 模式不要求电脑拥有公网 IP、端口映射或与手机处于同一网络。
- 安装包必须来自此私有仓库的 Release，并核对 Release 页面公布的 SHA-256。

## 安装或升级

1. 在手机上下载 `codeg-mobile-<版本>-universal.apk`。
2. 用文件管理器打开 APK；Android 首次安装时，按系统提示允许该来源安装应用。
3. 如果已经安装旧版，直接打开新 APK 覆盖升级。不要先卸载，否则本机配对凭据会被清除，需要重新配对。
4. 启动后确认应用名称为 **Codeg Mobile**。

电脑上可使用以下命令核对安装包：

```bash
shasum -a 256 codeg-mobile-<版本>-universal.apk
```

## 推荐：Relay 配对

1. 在电脑 Codeg 中打开 **设置 → 移动访问**。
2. 填写管理员提供的 Relay WSS 地址和桌面 Relay Token，启用移动访问。
3. 点击添加手机并生成一次性二维码。
4. 在 Codeg Mobile 首次连接页选择 **Relay**，允许相机权限并扫描二维码。
5. 手机和电脑会各显示一个六位安全码。确认两端完全一致后，只在电脑端点击确认。
6. 电脑确认后，手机自动进入工作区。后续启动不需要再次扫码。

二维码五分钟后过期，并且只能使用一次。已经完成或拒绝的二维码不能再次配对；不要通过聊天、邮件或截图转发二维码。

## Direct 连接

仅当手机能直接访问电脑或服务器的 HTTPS 地址时使用 Direct：

1. 在连接设置中选择 **Direct**。
2. 输入 Codeg HTTPS 地址和 Web Service Token。
3. 测试通过后保存并进入工作区。

Relay 长时间不可用时，断线提示中会出现 **切换 Direct**。Direct 仍然要求手机能访问该 HTTPS 地址。

## 日常使用与恢复

- 手机只负责创建任务、查看执行过程、批准权限、回答问题和上传附件；Agent、终端、Git 和项目文件仍运行在电脑端。
- 网络短暂中断、切换 Wi-Fi/移动网络或锁屏返回后，应用会自动重连并恢复事件游标。
- 如果五秒后仍未恢复，点击 **立即重连**；仍失败时检查电脑 Codeg、Relay 和手机网络。
- 未发送的输入不会跨应用重启、退出登录或服务器切换保留。
- 当前 Relay 附件上限为单文件 2 MiB；上传界面显示进度并支持取消，断线后会从桌面确认的分片位置继续。

## 撤销手机

在电脑 Codeg 的 **设置 → 移动访问 → 已配对设备** 中找到手机并撤销。撤销后旧凭据不能再次连接；需要恢复访问时必须生成新的二维码并重新确认。

## 隐私与安全

- Codeg Token、Relay 根密钥和设备私钥保存在系统安全存储中，不写入浏览器 Local Storage、URL 或日志。
- Relay 只转发端到端加密帧，无法读取代码、附件、聊天正文或 Codeg Token。
- Relay 仍可能观察连接时间、IP、设备路由标识和加密帧大小。
- 手机丢失时，应立即从电脑撤销该设备。

## 常见问题

### 一直显示“连接已断开”

确认电脑 Codeg 未退出或休眠、移动访问仍启用，并检查 Relay 健康状态。然后回到手机点击 **立即重连**。

### 扫码后电脑没有确认提示

二维码可能已过期或使用过。关闭旧配对窗口，重新生成二维码并在五分钟内扫描。

### 安装时提示签名冲突

手机上存在用另一把密钥签名的测试版。确认重要任务已结束后卸载测试版，再安装私有 Release；卸载会清除本机配对凭据。

### 上传失败

确认文件非空且不超过 2 MiB。网络中断后先等待自动恢复；必要时取消上传并重试。

## 当前已知限制

- 首个 Android Release 尚不包含 FCM 后台推送；应用被系统长期挂起时，需要手动回到前台恢复状态。
- 手机端不提供本地 Agent、终端、完整代码编辑器或 Git 操作。
- iOS TestFlight 版本独立交付，不包含在 Android APK 中。
