# Codeg Relay

Codeg Relay is a blind router for end-to-end encrypted Codeg Mobile frames. It
terminates WSS, authenticates routing credentials, limits traffic and forwards
opaque JSON envelopes between one desktop and its paired phones. It does not
receive the Codeg access token or decrypt Codeg requests, Agent events or files.

## Self-host with Docker

The recommended deployment is a small public VPS with a dedicated domain and
TLS termination. The desktop and every phone make outbound connections to it;
the desktop does not need a public IP, port forwarding or FRP.

```bash
cd relay/deploy
cp relay.env.example relay.env
openssl rand -hex 32
# Put the generated value in CODEG_RELAY_DESKTOP_TOKENS inside relay.env.
docker compose up -d --build
```

The container is published only on `127.0.0.1:18787`. Put
[`nginx-frp.conf`](deploy/nginx-frp.conf) inside an HTTPS virtual host or use
[`Caddyfile.example`](deploy/Caddyfile.example). Then configure the Codeg
desktop with `wss://relay.example.com/v1/ws` and the same desktop token.

The complete Chinese deployment, validation, FRP and migration guide is in
[`docs/relay/self-hosting-zh-CN.md`](../docs/relay/self-hosting-zh-CN.md).

## Run locally for development

```bash
export CODEG_RELAY_DESKTOP_TOKENS='{"d_example":"replace-with-at-least-32-random-characters"}'
export CODEG_RELAY_CREDENTIAL_FILE="$PWD/data/device-credentials.json"
cargo run --manifest-path relay/Cargo.toml
```

The process listens on `127.0.0.1:8787` by default. `ws://` is accepted only
for loopback development. Public desktop and mobile configurations must use a
valid `wss://` endpoint.

For a private single-user Relay where the desktop ID is generated by the app,
configure a wildcard token such as `{"*":"<random 32+ byte token>"}`. A
deployment serving multiple trust domains should instead provision one token
per explicit desktop ID.

Endpoints:

- `GET /health`
- `GET /metrics`
- `GET /v1/ws` followed by a five-second authenticated `hello`
- `POST /v1/devices` to issue a mobile routing credential
- `GET /v1/devices?desktop_id=...` to list paired credentials
- `DELETE /v1/devices/{device_id}?desktop_id=...` to revoke and disconnect

Device administration uses `Authorization: Bearer <desktop relay token>`.
Only SHA-256 mobile token hashes are persisted. The credential file is written
with mode `0600` on Unix.

Keep `/metrics` private, back up the credential volume, rotate the desktop
token if it is exposed, and re-pair phones after migrating to another Relay.
